选题判断

暂缓：补证搜索没有拿到可用结果，当前不能给可选或高可信判断。

ping0.cc 是广泛使用的 IP 纯净度查询工具，用户常在使用代理/VPN 时访问，期望隐藏真实 IP。该工具利用 WebRTC 绕过代理获取真实 IP 并记录，直接威胁用户隐私安全。此行为若属实，属于典型的隐私陷阱，对技术人、跨境工作者、独立开发者等群体具有普遍警示意义。

这件事目前能确认什么

核心问题：ping0.cc 是否通过 WebRTC 和双栈请求在用户不知情的情况下收集真实 IP，且该行为是否违反其隐私政策或相关法规？

  • V2EX 用户通过抓包发现 ping0.cc 的 check.js 中 peer() 函数利用 WebRTC 获取用户真实 IP，并通过 POST /ip/peer 发送至服务器。
  • 页面同时从 ipv4.ping0.cc 和 ipv6.ping0.cc 加载脚本，强制浏览器分别走 IPv4 和 IPv6，收集双栈用户的两个地址。
  • JS 代码中存在 GET /logv6/{ipv6}/{ip} 请求，将 IPv4 和 IPv6 配对记录到服务器。
  • 该行为在用户使用代理/VPN 时仍能获取真实 IP，且用户未被告知或同意。

线索提供了明确的技术细节和可复现步骤，逻辑自洽：用户使用代理访问 ping0.cc → 页面加载 JS → JS 通过 WebRTC 获取真实 IP → 通过 POST/GET 请求发送至服务器。双栈强制抓取进一步扩大了收集范围。该行为与用户期望（隐藏真实 IP）相悖，且未在用户界面或隐私政策中明确告知，构成隐私风险。

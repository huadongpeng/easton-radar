这件事目前能确认什么

核心问题：ping0.cc 是否通过 WebRTC 和双栈请求收集用户真实 IP 并配对存储？

  • 原帖作者通过浏览器 F12 抓包发现 ping0.cc 的 check.js 中存在 peer() 函数，利用 RTCPeerConnection 和 STUN 服务器获取用户真实 IP，并通过 POST /ip/peer 发回服务器。
  • 原帖作者发现 ping0.cc 从 ipv4.ping0.cc 和 ipv6.ping0.cc 两个子域名加载脚本，强制浏览器分别走 IPv4 和 IPv6 连接，从而获取双栈用户的两种地址。
  • 原帖作者发现 ping0.cc 通过 GET /logv6/{ipv6}/{ip} 将 IPv4 和 IPv6 地址配对发送到服务器。
  • Gridinsoft 对 ping0.cc 的信任评分为 59/100，提示存在安全风险。
  • WebRTC 泄露是已知的浏览器漏洞，多个网站（如 BrowserLeaks）提供检测和防护指南。

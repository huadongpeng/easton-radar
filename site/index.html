数据源原则

  • 优先官方、一手、近源和可复查资料。
  • 不把强反爬论坛、登录社区、截图爆料和纯营销软文作为核心源。
  • GitHub Actions 抓不到的反爬源及时放弃，不为了抓取炫技。
  • 任何入选选题必须保留来源、证据类型、可信度、疑点和不可夸大之处。
  • 每条报告必须先给出选题判断，再生成可交给 GPT 编辑应用和研究闭环的标准交接包。
  • 选题必须能回答：这事和一个懂一点技术、想提升收入但不想被割的人有什么关系。
  • 冷门技术产品只有在影响成本、岗位、平台规则、隐私安全、出海收入或可复制工作流时才进入深挖。
